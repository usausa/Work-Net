# Coding Style

- **General:** Follow the rules defined in .editorconfig
- **Instance field:** Do not use `_` prefix for member variables
- **Warnings:** Ensure there are no build warnings
- **Suppress warnings:** If warning suppression is needed, ask before applying the fix

# ドキュメント作成・技術調査

- **廃止サービス:** 廃止・非推奨、または新規受付を終了したサービスを推奨として提示しない。言及する場合は廃止日と現行の代替を明記する。
- **最新状況の確認:** 外部のサービス/SDK/モデルを推奨する前に、提供状況（提供可否・価格・後継）を学習データだけに頼らず確認する。確認した日付を記録する。
- **出典の明記:** 外部 API・サービスに関する記述には、公式/一次情報へのリンクを付ける。
- **不確実な点を断定しない:** 一次情報で仕様を確認できない場合は、その旨を述べ、確認先を示し、サンプルコードは雛形である旨を明記する。
- **作成日の明記:** 提供状況に依存して正確性が変わるドキュメントには、作成日や「○○時点」の注記を入れる。
- **比較してから推奨:** 単一の答えを押し付けず、選択肢を比較で示し、プロジェクトの制約に沿った推奨を添える。
- **意思決定の記録:** 検討して却下した選択肢とその理由をドキュメントに残す。
- **サンプルコードも規約準拠:** ドキュメントや README 内のコードも .editorconfig および本ファイルの規約に従う。
- **サンプルに機密を書かない:** エンドポイントやキーをコード・ドキュメントに直書きせず、環境変数や User Secrets から読み込む。
