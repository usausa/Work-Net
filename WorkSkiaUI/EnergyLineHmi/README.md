# EnergyLineHmi

WPF + SkiaSharp による **工場エネルギーライン監視 HMI**（俯瞰 P&ID ビュー）のサンプルです。
電力 / 都市ガス / 蒸気 / 復水・給水 の各系統を 1 枚のフロー図にまとめ、計測値とアニメーションでリアルタイム監視します。

![EnergyLineHmi のスクリーンショット](screenshot.png)

## 概要

- 仮想座標 1600×900 でレイアウトし、ウィンドウサイズへ等倍スケールして描画します。
- 全要素を `SKElement` に毎フレーム（`CompositionTarget.Rendering`）で描き直す、完全コードベースの計装図です。
- 実測機器の代わりに `PlantSimulation` がランダムウォーク＋一次遅れで計測値を生成します。

## 表示内容

| 区分 | 内容 |
|---|---|
| 配管 / 電力ライン | 流量に応じて流速・破線パターン・矢印が変化するアニメーション（電力＝黄、都市ガス＝青、蒸気＝橙、給水・復水＝シアン、冷却水＝緑、空気＝灰） |
| 機器ボックス | 受電点・変圧器・PV・ガスエンジン CGS・ボイラ・生産ライン・乾燥炉など。運転 / 停止 / 異常ランプと計測値を表示 |
| 母線・ヘッダ | 構内母線 6.6kV、蒸気ヘッダ圧力 |
| サイドパネル | 受電電力・構内発電・ガス使用量・蒸気供給のサマリ、受電デマンドゲージ（契約 2,000kW 比）、総消費電力トレンド（直近 90 秒）、アラーム / イベント履歴 |
| 計器 | 母線電圧・周波数・蒸気圧力のミニゲージ、給水タンク T-1 / 補給水タンク T-2 の水位 |

## 操作

| 操作 | 動作 |
|---|---|
| 機器ボックスをクリック | 起動 / 停止を切替（PV・CGS・ボイラ・生産ライン A/B・ユーティリティ・乾燥炉が対象） |
| ホバー | 操作可能な機器を強調表示 |

運転中の機器にはランダムな擬似異常が発生し、数秒後に自動復旧します。受電デマンド超過・給水タンク低水位・蒸気供給不足はプロセス警報としてイベント履歴に記録されます。

## 実行方法

```powershell
dotnet run -c Release
```

> 要 .NET 8 SDK / Windows（WPF）。

## 構成

```
MainWindow.xaml(.cs)        SKElement の配置とマウス操作・再描画ループ
Rendering/HmiRenderer.cs    フロー図本体（配管・機器ボックス・凡例）とヒットテスト
Rendering/HmiPanels.cs      ヘッダ・サイドパネル（KPI / デマンドゲージ / トレンド / アラーム）
Rendering/HmiElements.cs    タンク・ポンプ・ファン・冷却塔・アナログ計器
Rendering/Theme.cs          配色・フォント定義
Simulation/PlantSimulation.cs  各計測値のシミュレーションと警報ロジック
```
