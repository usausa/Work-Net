# WorkML

電力・電圧の保安機器ログを用いた **予防保守（予知保全）** の検討用ワークスペース。
Azure の AutoML 系サービスと ML.NET を **.NET から** 使う前提で、選択肢・アーキテクチャ・サンプル仕様を整理する。

## ドキュメント
- [保安機器ログを用いた予防保守 — Azure AutoML 系 × .NET 検討ドキュメント](docs/PredictiveMaintenance-AutoML.md)

## 要点（3行）
- **異常検知（ML.NET / ラベル不要）から始め**、故障履歴が貯まったら **Azure ML AutoML で故障予測**へ。
- 本番推論は **AutoML 学習 → ONNX → ML.NET ローカル推論** が本命（低レイテンシ・オフライン可）。REST エンドポイントは集中管理向け。
- **異なる基本電圧は標幺値(p.u.)正規化**で吸収し、**可変チャンネルは「系列(grain)」または「集約特徴」**でモデル化する。

> ⚠️ Azure AI Anomaly Detector は 2026/10/01 廃止（新規作成不可）のため採用しない。詳細はドキュメント参照。
